Tecnologias: Frontend

Next.js & Ecossistema

Por que utilizar?

  • Next.js & React: Next.js é um framework React que oferece renderização no servidor (SSR), melhorando a performance e o SEO. React permite criar interfaces componentizadas e reutilizáveis.
  • TypeScript: Adiciona segurança de tipos ao JavaScript, o que reduz erros, melhora a previsibilidade do código e facilita a colaboração em equipe.
  • Tailwind CSS & Framer Motion: Tailwind acelera a estilização com classes utilitárias, enquanto Framer Motion permite criar animações fluidas, melhorando a experiência do usuário.

Possível Alternativa

Vue.js + Nuxt.js: Uma stack igualmente poderosa, conhecida por sua curva de aprendizado suave e excelente documentação.

Tecnologias: Backend

Node.js, Express & Ferramentas

Por que utilizar?

  • Node.js & Express: Criam um backend rápido, flexível e minimalista. Manter o JavaScript/TypeScript em toda a stack unifica a linguagem e facilita o desenvolvimento.
  • Prisma ORM: Simplifica e torna segura a comunicação com o banco de dados, usando TypeScript para autocompletar e validar queries, prevenindo erros comuns de SQL.
  • Multer: É um middleware essencial e robusto para o upload de arquivos. Ele gerencia os dados de formulários `multipart/form-data`, facilitando o recebimento e armazenamento dos currículos.

Possível Alternativa

TypeORM: Outro ORM popular para TypeScript, com uma abordagem mais tradicional baseada em classes e decoradores.

Tecnologias: Processamento de Arquivos

pdf-parse & mammoth.js

Por que utilizar?

  • Leves e Rápidas: Bibliotecas focadas e otimizadas para uma única tarefa, garantindo performance.
  • Integração Simples: Facilmente integráveis ao nosso backend Node.js, sem dependências externas.
  • Custo Zero: Soluções open-source que não adicionam custos ao projeto.

Possível Alternativa

Apache Tika: Uma solução mais robusta que suporta centenas de formatos, mas exige uma infraestrutura mais complexa (geralmente via API).

Tecnologias: Infraestrutura

Vercel, Railway & GitHub

Por que utilizar?

  • Deploy Simplificado: Vercel (frontend) e Railway (backend) oferecem integração contínua (CI/CD) via GitHub.
  • Foco no Desenvolvedor: Abstraem a complexidade da infra, permitindo que a equipe foque no código.
  • Escalabilidade e Custo: Planos gratuitos generosos e escalabilidade automática conforme a demanda.

Possível Alternativa

AWS / Google Cloud: Soluções mais poderosas e customizáveis, mas com uma curva de aprendizado e complexidade de configuração maiores.

Segurança: Autenticação com JWT

JSON Web Token

Por que utilizar?

  • Stateless: O servidor não precisa armazenar o estado da sessão, facilitando a escalabilidade.
  • Seguro: Os tokens são assinados digitalmente, garantindo a integridade dos dados e a autenticidade do remetente.
  • Padrão de Mercado: Amplamente adotado e com suporte em diversas linguagens e frameworks.

Possível Alternativa

Sessões baseadas em Cookies: Uma abordagem mais tradicional, onde o servidor armazena os dados da sessão, mas pode ser mais complexa de escalar.

Documentação: Swagger (OpenAPI)

Swagger UI

Por que utilizar?

  • Documentação Interativa: Permite que os desenvolvedores testem os endpoints da API diretamente pelo navegador.
  • Fonte Única de Verdade: A documentação é gerada a partir do próprio código, garantindo que esteja sempre atualizada.
  • Facilita a Integração: Clarifica o "contrato" da API, acelerando o desenvolvimento paralelo entre frontend e backend.

Possível Alternativa

Postman Documentation: Outra ferramenta excelente, mas que exige a manutenção manual da coleção de requisições.

Ferramentas para Organização

Para garantir a visibilidade e o alinhamento do projeto, utilizaremos ferramentas de organização.

Jira

Ideal para um planejamento detalhado de Sprints, gestão de backlog, e acompanhamento de bugs. Oferece relatórios avançados para medir a produtividade da equipe. Ideal se tiver expansão da equipe.

Trello

Perfeito para a gestão do dia a dia com seu sistema de quadros Kanban. Sua simplicidade visual facilita o acompanhamento rápido do status de cada tarefa. Nossa escolha pelo tamanho da equipe, não precisa mais que isso.
